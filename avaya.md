## Avaya Communication (2001 - 2005) -- Lead Architect

### Led Development of Major Product -- MVAP
- **Supervised 20-person** system architecture development team
- Led software development of what became the **2nd highest selling product in Avaya's history**
- For a **billion-dollar manufacturer** of PBX and call center systems
- Product: **MVAP** -- an application server that opened up an API into Avaya's legacy switches; applications could run remote or embedded in the application server
- Oversaw development and fixes; delivered presentations to clients
- **Defined APIs into 5 architectural components**; invented open source ant task to guarantee APIs don't depend on developer implementations

### Process & Team Improvements
- Introduced new testing, verification, and automation processes to enhance efficiency and quality
- **Re-organized team structure** to emphasize object-oriented processes
- **Convinced organization to adopt automated builds** using CruiseControl (when it first came out)
- **Influenced team to write first JUnit tests** on the project
- Mentored developers into being mentors for others by getting individuals to present topics
- API changes were mailed to Dean so he could track how architecture was evolving
